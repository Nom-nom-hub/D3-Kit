# /d3.quickstart - Create Validation Guide

## Execute the Script

**On Linux/Mac:**
```bash
bash scripts/bash/d3-quickstart.sh "d3-features/001-feature-name"
```

**On Windows:**
```powershell
. scripts/powershell/d3-quickstart.ps1 "d3-features/001-feature-name"
```

## What This Does

1. Validates that spec.md exists
2. Creates `quickstart.md` file
3. Provides structure for:
   - Prerequisites and setup steps
   - How to run/access the feature
   - Validation scenarios (happy path, edge cases)
   - Troubleshooting guide
   - Performance baseline expectations
   - Security validation checklist
   - Success criteria verification

## Output

Creates file: `d3-features/001-feature-name/quickstart.md`

## How to Use

1. Run `/d3.quickstart` to generate the template
2. Write setup instructions for developers
3. Document how to run/access the feature
4. Create validation scenarios to test
5. Add troubleshooting tips
6. Define expected performance metrics
7. Create security validation checklist
8. List success criteria to verify

## When to Run

- After implementation plan is created
- Before starting implementation
- Used during testing/validation phase
